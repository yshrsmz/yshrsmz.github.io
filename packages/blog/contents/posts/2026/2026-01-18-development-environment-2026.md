## OS

OS は Mac を使っている。会社が Mac なので、コンテキストスイッチを減らすために個人用も Mac。最近会社的には Windows でも良さそうなんだけど、iOS 開発もあることを考えると Mac 一択になる。

家ではたまに Windows のソフトウェアを使いたくなることもあるんだけど、そんなときのために Parallel Desktop を用意している。そして PC ゲームは Steam Deck。

Mac は Apple Silicon になってからバッテリー性能が大変優秀なので、そういう意味でも好ましい。Windows も ARM 版が普及してハイスペック機も出てきたら選択肢に上がりそう。今はWSLもあるし。

## エディタ

エディターは VSCode と Android Studio、たまにIntelliJ IDEAを使っている。基本はVSCodeで、Android や KMP をやるときは Android Studio を使い、Gradle Plugin を書くときは IntelliJ IDEA を使う。適材適所。

Cursor はちょっと試したけど、そもそも AI エージェントは Claude Code をメインで使ってるからわざわざ Cursor にする必要がなかった。あと Cursor は DevContainer 周りが VSCode と違う拡張機能を使うことになり、それがまあ微妙な使い心地なのでこれを嫌ったというのもある。

## シェル

シェルは zsh で、特にこだわりはない。社会人になって zsh がイケてるぞと言われて使い始め、そのまんま使っている感じ。カスタマイズもあまりしていなくて、peco と ghq でローカルリポジトリの検索と遷移ができるようにしたり、コマンド実行履歴の検索できるようにしているくらい。

## ターミナル

最近は [Ghostty](https://ghostty.org/) を使っている。そのちょっと前は WezTerm で、さらにその前はずっと iTerm2 を使っていた。WezTerm も良かったんだけど、なんかウィンドウをドラッグしようとすると、掴んで動かそうとした瞬間にウィンドウが掴んだ位置からずれるというよくわからん挙動するのが地味に気になっていた。

Ghostty は数年前に試したことがあって、そのときは Cmd+F での検索ができなくて離脱してしまった。しかし最近 Cmd+F での検索機能が追加されて tip 版で使えるようになったので再び試してみている。軽快に動くし、WezTerm のようにドラッグ時の挙動が変、ということもないので大変満足。


## コーディングエージェント/AI

メインで [Claude Code](https://claude.com/product/claude-code) を使っている。MAX 200 ドルプラン。

その他だと [Codex CLI](https://developers.openai.com/codex/cli/) も使っていて、これは基本的には Claude Code の MCP として動かしている。Codex 遅いのでさすがにメインで使う気にはならないけど、やる内容の精度は高いから Claude Code に書かせたプランやコードのレビュー用途では非常に役立っている。
よく使っている MCP は、上述の Codex MCP, Serena, Context7 くらい。Context7 は使っているというか、設定はしているもののあんまり動いているのを観測したことはないので消してもいいかもしれない。

spec-driven development 系は、spec-kit と spec-workflow-mcp, [cc-sdd](https://github.com/gotalab/cc-sdd) を試して、今のところ cc-sdd がよさそう。spec-workflow はダッシュボードが GUI でわかりやすいのが強み。一方で cc-sdd は仕様書や設計書の検証系ツールが充実している。作成した仕様書と既存実装の差分とか、実装が要件/設計に合致しているかとか検証できるのはありがたい。あとは生成されるドキュメントも cc-sdd のほうがコンパクトでよろしい。

生成されるドキュメント類に関しては、最近は日本語で出力してもらったほうが読みやすいし結果的に精度上がるな、と考えている。初期は「英語読めるし英語で書いてもらったほうが正確だしコンテキストも節約できるでしょ」と思っていたけど、やっぱり第一言語ではないので疲れるし目が滑るから見逃しからの修正が多いことに気づいた。

## ブラウザ

ブラウザは基本的にChromeを使っている。Androidを使っているし、Google系のサービスにロックインされているので、Googleのサービスが一番使いやすいブラウザを使い続けている。
ただ、同じ使いやすさみたいな観点でいくと、最近異動した子会社のグループウェアがMS365なので、そのアカウントは MS Edge で開いている。

## キーボード/トラックボール

最近一番使っているのは [JezailFunder の Cornix](https://jezailfunder.jp/products/cornix-lp-keyboard)。40%で左右分割、ワイヤレスなキーボード。Cornix は無線キーボードでありながら Vial を利用することができる点が、他のZMKを採用した無線キーボードと違う圧倒的なアドバンテージだと思う。GUIで簡単にコンボやタップダンスまでいじれるのは本当に楽。

ポインティングデバイスは [Kensington SlimBlade Pro](https://www.kensington.com/ja-jp/p/%E8%A3%BD%E5%93%81/%E3%82%B3%E3%83%B3%E3%83%88%E3%83%AD%E3%83%BC%E3%83%AB/%E3%83%88%E3%83%A9%E3%83%83%E3%82%AF%E3%83%9C%E3%83%BC%E3%83%AB/slimblade-pro%E3%83%88%E3%83%A9%E3%83%83%E3%82%AF%E3%83%9C%E3%83%BC%E3%83%AB3/)。無線で使うこともできるけど、Bluetooth モードはペアリング一台しかできないし、USBドングルのWiFiモードだと定期的に接続が切れるので、不本意ながら有線で繋いでいる。トラックボール付きのキーボードも考えて入るんだけど、市場にあるのは親指トラックボールばかりなので悩ましい。親指トラックボール使ってて腱鞘炎になりかけたことがあるのでちょっと敬遠している。

あと入力デバイスという括りでいくと、[Aqua Voice](https://aquavoice.com/) という AI 音声入力ツールの精度がとても高くて捗る。AI エージェントへの入力を音声で済ませることができるし、それ以外にも文章の草案を口頭でまとめて推敲したりとか。この文章も Aqua Voice でざっと喋ったものを AI にアウトラインまとめさせてから書いている。自分が喋るの苦手ということもあり音声入力に今まで懐疑的だったんだけど、やはり口に出してみると自分の中で色々整理されて考えがまとまっていくものだなあ、というのを Aqua Voice を使いながら日々実感している。

[1ヶ月間 Pro プランで使える紹介リンク](https://aquavoice.com/share?code=YS-1EA6)を置いておくので、ぜひ試してみてほしい。

## メモ/ノートアプリ

現在は [Obsidian](https://obsidian.md/) を活用している。その前はNotionを使っていたんだけど、コーディングエージェントを使うようになって、AIから一番使いやすいツールって考えたときに、ローカルにテキスト形式で全てのメモを残せる Obsidian が第一の選択肢に挙がった。直近で Bases っていうデータベースっぽい仕組みが入ったりしたので、だいぶ Notion のような使い方もできるようにはなってきた。

ローカルにファイルがあるので、Marp を Obsidian の Vault 内にセットアップして Markdown でプレゼン資料書いたりもできる。

## Dotfiles/プロビジョニング/環境構築

この辺は10年くらいずっと Ansible で管理していた。が、PC移行の度に毎回 Ansible のバージョンアップ作業から始まるのがいい加減めんどくさくなったので、去年思い切って [go-task/task](https://github.com/go-task/task) とシェルスクリプトの組み合わせに移行した（これも Claude Code にシュッとやらせた）。

アプリケーションの管理は [Homebrew](https://brew.sh/)。開発環境の管理ツールは [mise](https://mise.jdx.dev/)。去年までは anyenv 経由で nodenv やら rubyenv やら使ってたけど、異動先の構成に合わせて mise に意向。mise.toml に対応した GitHub Actions がまだ少ないのはちょっとめんどい。

その他、最近は可能であれば必ず devcontainer を設定するようになった。開発環境の標準化というのもあるけど、一番はやはり AI を安全に動かせる環境を用意するためだ。ただ Android や iOS 周りはまだ仮想化厳しいのでなんとかなってほしい。

## その他

ランチャーアプリはずっと [Alfred](https://www.alfredapp.com/) を使っている。一瞬 Raycast も試したんだけど、例えば辞書検索するときとか、Alfred だと「define foobar」って打つとそのまま辞書の検索ができるんだけど、Raycast だと辞書のホットワードを打ち込んだ後にエンターを押さないと辞書モードにならないとか、ちょっとしたところで Alfred よりひと手間が多いところが気になってしまった。ただ設定でどうにかなるのかとか試してないし、今どっちが活発かって言ったらレイキャストの方が活発なのはそれはそうなのでもう一回試してみるのもいいかも。
